# Screenshots Directory

Place your demonstration screenshots here with the following names:

## Required Screenshots

1. **hot_reload_demo.png**
   - Shows before/after of a Hot Reload change
   - Should demonstrate instant UI update
   - Include code editor and running app

2. **debug_console.png**
   - VS Code with Debug Console visible
   - Should show multiple debugPrint() outputs
   - Highlight the log messages with emojis

3. **widget_inspector.png**
   - DevTools Widget Inspector tab
   - Widget tree visible on left
   - Properties panel on right
   - Selected widget highlighted in app

4. **performance_view.png**
   - DevTools Performance tab
   - Frame rendering timeline
   - Should show green bars (good performance)

5. **memory_profiler.png** (Optional)
   - DevTools Memory tab
   - Memory usage graph
   - Snapshot comparison

6. **network_tab.png** (Optional)
   - DevTools Network tab
   - HTTP requests if applicable
   - Request/response details

7. **full_devtools.png**
   - Complete DevTools interface
   - Shows all available tabs
   - Overview of capabilities

## Screenshot Guidelines

- **Resolution:** Minimum 1280x720, recommended 1920x1080
- **Format:** PNG (lossless quality)
- **Content:** Clear, readable text and UI elements
- **Cropping:** Include relevant context, remove excess whitespace
- **Annotations:** Add arrows or highlights if needed (optional)

## Taking Screenshots

### Windows
- **Snipping Tool:** Windows + Shift + S
- **Full Screen:** Windows + PrtScn
- **ShareX:** Advanced tool with annotations

### Mac
- **Selection:** Cmd + Shift + 4
- **Window:** Cmd + Shift + 4, then Space
- **Full Screen:** Cmd + Shift + 3

### VS Code
- Install "Polacode" extension for beautiful code screenshots

## After Capturing

1. Name files exactly as listed above
2. Save to this `screenshots/` directory
3. Reference in README.md using:
   ```markdown
   ![Description](screenshots/filename.png)
   ```
4. Commit screenshots with your code changes
